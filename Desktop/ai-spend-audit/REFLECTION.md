# Project Reflection

## 1. Hardest Bug
The challenge of handling "Mixed Tiers" (e.g., a company with 5 users on Pro and 2 users on Enterprise). I solved this by normalizing all inputs into a "cost-per-seat" object before running the comparison.

## 2. Mid-week Reversal
Originally, I planned to use a heavy SQL backend. I reversed this to use a lightweight Supabase setup to prioritize the UI polish and "Viral Loop" features.
